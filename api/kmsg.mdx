The `kmsg` package contains code-generated structs for every request and response in the Kafka wire protocol. You use it when the high-level `kadm` package does not expose the operation or level of detail you need. Every request type has a `RequestWith` method that issues the request through any `kgo.Client` or `kgo.Broker` and returns a typed response. The package follows strict versioning rules: always use the `New*` constructor functions to initialize request structs so that new fields get their correct default values.

```bash
go get github.com/twmb/franz-go
```

```go
import "github.com/twmb/franz-go/pkg/kmsg"
```

For the complete auto-generated reference, see [pkg.go.dev/github.com/twmb/franz-go/pkg/kmsg](https://pkg.go.dev/github.com/twmb/franz-go/pkg/kmsg).

## When to use kmsg vs kadm

Use `kadm` for everyday admin work: creating topics, managing offsets, describing configs. Use `kmsg` directly when:

- You need fields or request variants that `kadm` does not expose.
- You want to issue requests to a specific broker rather than the controller or coordinator.
- You are building tooling that needs access to every field in the Kafka protocol response.
- You need to issue requests inside `kfake` control functions to simulate specific broker behavior.

## Always use New\* constructors

The kmsg package may add new fields when Kafka introduces new protocol versions. A new field that has a non-zero default value will behave incorrectly if you initialize the struct with a Go literal. Always use the constructor:

```go
// Correct: safe defaults for all fields.
req := kmsg.NewMetadataRequest()
req.Topics = nil // fetch metadata for all topics

// Incorrect: new fields with important defaults may be zero-valued.
req := kmsg.MetadataRequest{}
```

## RequestWith pattern

Every generated request type has a `RequestWith` method that accepts any `kmsg.Requestor`. Both `*kgo.Client` and `*kgo.Broker` implement this interface:

```go
req := kmsg.NewMetadataRequest()
resp, err := req.RequestWith(ctx, client)
if err != nil {
    log.Fatal(err)
}
for _, t := range resp.Topics {
    fmt.Printf("topic=%s partitions=%d\n", t.Topic, len(t.Partitions))
}
```

## Issuing requests with kgo.Client

`kgo.Client` provides two methods for raw protocol requests:

| Method | Description |
|---|---|
| `cl.Request(ctx, req)` | Issues a request to any broker (or the controller for admin requests). Returns a `kmsg.Response`. |
| `cl.RequestSharded(ctx, req)` | Splits the request across brokers and returns a `[]kgo.ResponseShard`. Use this for requests that fan out, like `ListOffsets`. |

```go
req := kmsg.NewListOffsetsRequest()
req.Topics = []kmsg.ListOffsetsRequestTopic{{
    Topic: "my-topic",
    Partitions: []kmsg.ListOffsetsRequestTopicPartition{{
        Partition: 0,
        Timestamp: -1, // latest
    }},
}}

shards := cl.RequestSharded(ctx, req)
for _, shard := range shards {
    if shard.Err != nil {
        log.Printf("broker %d: %v", shard.Meta.NodeID, shard.Err)
        continue
    }
    resp := shard.Resp.(*kmsg.ListOffsetsResponse)
    for _, t := range resp.Topics {
        for _, p := range t.Partitions {
            fmt.Printf("partition=%d offset=%d\n", p.Partition, p.Offset)
        }
    }
}
```

```go

kmsg.ApiVersions      // Key = 18
```

The constant `kmsg.MaxKey` holds the highest defined API key.

## Core interfaces

| Interface | Description |
|---|---|
| `Request` | Implemented by all request types. Provides `Key()`, `MaxVersion()`, `AppendTo()`, `ReadFrom()`, and `ResponseKind()`. |
| `Response` | Implemented by all response types. Mirror of `Request`. |
| `Requestor` | Accepts a `Request` and returns a `Response`. Implemented by `*kgo.Client` and `*kgo.Broker`. |
| `AdminRequest` | Extends `Request`; marks requests that must go to the controller broker. |
| `GroupCoordinatorRequest` | Extends `Request`; marks requests that must go to a group coordinator. |
| `TxnCoordinatorRequest` | Extends `Request`; marks requests that must go to a transaction coordinator. |

<Note>
The `MaxKey` constant and the set of `Key` variables are the authoritative list of API keys. The generated code in `generated.go` defines one `New*` constructor and one `NewPtr*` constructor per request/response type.
</Note>

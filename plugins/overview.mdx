franz-go separates observability concerns from the core client through a hook-based plugin system. The `kgo.Client` exposes a set of interfaces — collectively called hooks — that plugins implement to receive callbacks at key points in the client lifecycle: when connections open or close, when bytes are written or read, when batches are produced or fetched, and more. Because each plugin is an independent Go module, you only pull in what you need.

## How plugins work

Plugins implement one or more hook interfaces from the `kgo` package. You register them when constructing the client using `kgo.WithHooks` (for metrics) or `kgo.WithLogger` (for logging). The client checks at runtime which interfaces each registered hook satisfies and calls only the methods that exist on it — your code is never forced to implement methods it does not need.

```go
import (
    "github.com/twmb/franz-go/pkg/kgo"
    "github.com/twmb/franz-go/plugin/kprom"
    "github.com/twmb/franz-go/plugin/kzap"

    "go.uber.org/zap"
)

metrics := kprom.NewMetrics("myapp")
zapLogger, _ := zap.NewProduction()

cl, err := kgo.NewClient(
    kgo.SeedBrokers("localhost:9092"),
    kgo.WithHooks(metrics),
    kgo.WithLogger(kzap.New(zapLogger)),
)
```

Metrics hooks and logging hooks are independent: you can combine any logger with any metrics plugin, or use multiple metrics plugins together.

## Available plugins

Each plugin lives under `github.com/twmb/franz-go/plugin/{name}` and must be installed with a separate `go get` call.

## Module path pattern

All plugins follow the module path `github.com/twmb/franz-go/plugin/{name}`. Because each plugin is its own Go module with its own `go.mod`, adding one plugin does not pull in the dependencies of any other. Install only what you use:

```bash
# Install the Prometheus metrics plugin
go get github.com/twmb/franz-go/plugin/kprom

# Install the slog logging plugin
go get github.com/twmb/franz-go/plugin/kslog
```

## Writing a custom hook

If none of the built-in plugins suit your needs, you can implement the hook interfaces directly. The `kgo.Hook` type is `any`, so any struct that implements one or more hook interfaces qualifies. See the `HookBrokerConnect`, `HookBrokerWrite`, `HookProduceBatchWritten`, and related interfaces in the `kgo` package for the full list of available hooks.

```go
type myMetrics struct{}

// OnBrokerConnect is called each time the client opens a connection to a broker.
func (m *myMetrics) OnBrokerConnect(meta kgo.BrokerMetadata, _ time.Duration, _ net.Conn, err error) {
    if err != nil {
        // increment an error counter
        return
    }
    // increment a connect counter
}

cl, err := kgo.NewClient(
    kgo.SeedBrokers("localhost:9092"),
    kgo.WithHooks(&myMetrics{}),
)
```
